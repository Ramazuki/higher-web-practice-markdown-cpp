# MarkdownToHTML Converter
___

## Структура проекта

Проект разделён на две части:

- Приложение - `main.cpp`
- Библиотека конвертера - подпроект `converter`

В папке `converter` находятся файлы, в которые необходимо добавить свой код конвертера.
Непосредственно приложение, использующее конвертер, должно быть написано в файле `main.cpp` и должно использовать написанный конвертер.

## Сборка и запуск программы

Сборка и запуск программы осуществляется локально, для этого используется `cmake`.
Для сборки при помощи `cmake` необходимо иметь файл конфигурации `CMakeLists.txt` - он дан в прекоде. Сборка осуществляется в два этапа, командами, которые надо вызывать из корня проекта:
- `cmake -B build -S .` - конфигурирование сборки в папку `build`, использование контекста текущей директории `./`
- `cmake --build build --parallel` - сборка проекта, сконфигурированного в папке `build`, параллельная сборка с использованием всех ядер процессора (для ускорения сборки)
  При успешной сборке у вас появится исполняемый файл `build/MarkdownToHTML`.
- Для запуска программы выполните команду `./build/MarkdownToHTML --in examples/input.md` из корня проекта.

Для упрощения сборки и запуска в прекода также даны скрипты:
- `build.sh` - скрипт для конфигурации и сборки
- `run.sh` - скрипт для запуска проекта
- `build_and_run.sh` - скрипт для сборки и запуски проекта "в одно движение"

Данные скрипты следует запускать из корня проекта.
